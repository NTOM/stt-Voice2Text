
你 **必须** 在执行前考虑用户输入（如果非空）。

## Operating Constraints

- **文件写入范围**：仅允许写入 `Export/` 目录下的 `.md` 文件
- **禁止创建脚本**：严禁创建任何临时 Python 脚本（`.py`）、文本文件（`.txt`）、JSON 文件等中间产物。Token 估算、幂等性检查、内容提取等操作必须由 AI 自身内联完成（读取文件 + 直接计算），不得写脚本执行
- **禁止调用外部 LLM**：AI 重写必须由当前 IDE 内的 AI 自身直接生成，严禁编写脚本调用任何外部 LLM API（如 OpenAI、Anthropic 等接口）
- **幂等性保证**：重复执行同一输入必须产生一致结果，不会产生重复内容
- **渐进式加载**：原文 > 20000 tokens 时分段重写后合并
- **错误即停**：任何步骤失败必须立即报告并停止，不得静默跳过
- **上游隔离**：不修改 `tools/json2md.py` 和任何上游文件

## Outline

### 1. 输入验证与脚本调用（提取原文）

**输入验证**：

1. 检查 `{{json-file-path}}` 是否为非空字符串
   - 为空 → 提示用户提供 JSON 文件路径，**ABORT**
2. 检查文件是否存在
   - 不存在 → 提示用户检查路径，**ABORT**
3. 检查文件扩展名是否为 `.json`
   - 非 `.json` → 提示文件格式不正确，**ABORT**
4. 读取文件前几行，验证是否为有效 JSON（包含 `[` 或 `{` 开头）
   - 无效 → 提示 JSON 格式异常，**ABORT**
5. 检查文件编码是否为 UTF-8
   - 非 UTF-8 → 提示编码问题，建议转换为 UTF-8，**ABORT**

**脚本调用**：

运行 Python 脚本将 JSON 转录文件转换为 Markdown：

```bash
python tools/json2md.py "{{json-file-path}}" -f
```

> `-f` 标志确保覆盖已有输出，保证幂等性。

**错误处理**：
- 退出码 ≠ 0 → 输出错误信息，**ABORT**（不继续后续步骤）
- 输出文件为空 → 提示 JSON 内容可能异常，**ABORT**

记录生成的 `.md` 文件绝对路径（后续步骤引用为 `OUTPUT_MD`）。

### 2. 读取原文与 Token 估算

1. 读取 `OUTPUT_MD` 文件内容
2. 提取正文内容：定位 `## 第一章：原文` 标题，提取其后的全部正文（不含元数据头部）
3. **保存元数据头部**：记录文件开头到 `## 第一章：原文` 标题之前的内容（含标题、Source、Segments、Duration 等），后续覆盖写入时需保留
4. 估算 Token 数量：
   - 中文字符数 × 1.5（每字符 ≈ 1.5 tokens）
   - 英文单词数 × 1.0（每单词 ≈ 1 token）
   - **总 Token 数** = 中文 tokens + 英文 tokens
5. 记录：估算 token 数、总字符数、段落数

**阈值判定**：20000 tokens

| 条件 | 路径 | 加载策略 |
|------|------|----------|
| ≤ 20000 tokens | 路径 A（全量重写） | 一次性加载全文 |
| > 20000 tokens | 路径 B（分段重写） | 按段落组渐进加载 |

**错误处理**：
- 未找到 `## 第一章：原文` 标题 → 提示文件格式异常，**ABORT**
- 正文内容为空 → 提示转录内容可能为空，**ABORT**

### 3. 幂等性检查

由于最终输出为单一文档（覆盖写入），天然幂等——重复执行同一输入始终产生一致结果，无需额外检查。

直接进入 Step 4。

### 4. AI 重写（生成最终稿）

#### 路径 A：全量重写（≤ 20000 tokens）

对提取的原文执行一次性 AI 重写，遵循以下指令：

> **AI 重写 Prompt：**
>
> 你是一位专业的演讲稿编辑。请对以下口语转录原文进行重写优化，生成一份结构清晰、可读性强的演讲稿。
>
> ## 重写规则（必须全部遵守）：
>
> ### 1. 子标题分配
> - 使用 `###` 三级标题（`##` 已用于章节标题）
> - 采用**内容概括式命名**（如 `### Harness Engineering 的定义`、`### 三层演进链条`）
> - **不带编号**（不使用 `### 1.`、`### 第一节` 等形式）
> - 根据内容语义自然分段，每个子标题下覆盖一个完整的话题
>
> ### 2. 标点符号补充
> - 补充合理的标点符号：句号（。）、逗号（，）、问号（？）、感叹号（！）、顿号（、）等
> - 使语句通顺、断句合理
> - 使用中文标点符号
>
> ### 3. 重新分段
> - 将连续的长文本按语义重新分段
> - 每段聚焦一个要点，段落间留空行
> - 避免单段过长（建议每段 100~300 字）
>
> ### 4. 对话性内容检测
> - 识别**互动性内容**：弹幕互动（如「扣个1让我看看」）、向观众提问（如「你们有没有遇到过...」）、号召行动（如「记得点赞关注」）
> - 将这些内容使用 Markdown 引用块（`>`）格式标注，与叙述性正文明确区分
> - 如果全文无互动性内容（如纯技术讲座），则跳过此步骤，全部作为叙述性内容处理
>
> ### 5. 风格保持
> - **保持演讲稿/口语的自然风格**，不过度书面化
> - 保留原文的语气词和表达习惯（适度保留，不全部删除）
> - 不得添加原文中不存在的观点或信息
> - **保留原文 ≥ 95% 的核心信息点**
>
> ## 输出要求：
> - 直接输出重写后的内容，不需要额外的说明或解释
> - 使用中文输出
>
> ---
>
> 以下是需要重写的原文内容：
>
> {Step 2 提取的正文内容}

#### 路径 B：分段重写（> 20000 tokens）

**渐进式加载策略**：每次仅加载一个段落组到上下文，重写完成后释放，再加载下一组。

1. **信息性提示**（非阻塞，自动继续）：
   ```
   ℹ️ 内容较长（约 N 字 / ~T tokens），将分 M 段进行渐进式重写
   ```

2. **按语义段落分组切分**：
   - 将段落按顺序分组，每组 ≤ 15000 tokens
   - **绝不在段落中间切分** — 始终在段落边界处切分
   - 记录总段数 M

3. **逐段渐进重写**（每次仅加载当前段）：

   > 这是第 X/M 段内容。请按照上述重写规则（子标题分配、标点补充、重新分段、对话检测、风格保持）对以下内容进行重写。
   >
   > {当前段落组内容}

4. **合并重写结果**：
   - 按原文顺序拼接各段重写结果
   - 检查段落衔接处是否自然，必要时微调过渡
   - 确保子标题不重复、层级一致

### 5. 覆盖写入最终文档

用 AI 重写内容 **直接覆盖** `OUTPUT_MD` 文件，最终文件结构为：

```markdown
{Step 2 保存的元数据头部}

---

## 原文

{Step 4 生成的重写内容（### 三级子标题 + 正文段落）}
```

> **注意**：不保留 `## 第一章：原文` 标题和原始转录文本。最终文件仅包含元数据头部 + 分隔线 + `## 原文` 二级标题 + AI 重写后的正文。`## 原文` 作为统一的二级标题，与后续 `stt.summarize` 命令生成的 `## 内容分析` 形成对称结构。

### 6. 完成报告

**输出验证**（自动执行，不阻塞）：

- [ ] `OUTPUT_MD` 文件存在且非空
- [ ] 包含元数据头部（Source、Segments、Duration）
- [ ] 包含至少一个 `###` 三级子标题
- [ ] 包含 `## 原文` 二级标题（统筹所有正文内容）
- [ ] 不包含 `## 第一章：原文` 标题（已被覆盖）

**完成报告**：

```
✅ 转换完成

| 指标 | 值 |
|------|-----|
| 输出文件 | {OUTPUT_MD 路径} |
| 原文字符数 | {N} 字 |
| 估算 Token 数 | {T} tokens |
| 重写路径 | 路径 A（全量） / 路径 B（分 M 段） |
| 子标题数 | {K} 个 |

📄 单一文档输出（`## 原文` 统筹正文，与 `## 内容分析` 对称）
```

**错误处理**：
- 验证项任一失败 → 输出失败项 + 建议修复操作，标记为 ⚠️ 部分完成

## Behavior Rules

- **绝不跳过错误**：任何步骤失败必须立即停止并报告，不得静默继续
- **绝不重复执行脚本**：`json2md.py` 在整个流程中仅执行一次
- **绝不创建中间文件**：整个流程中不得创建任何 `.py`、`.txt`、`.json`、`.md`（`OUTPUT_MD` 除外）等临时文件。所有计算（Token 估算、幂等性检查、内容分析）均由 AI 直接在对话中完成
- **AI 自身生成重写内容**：重写内容由当前 IDE 的 AI 直接生成并写入文件，不得编写脚本调用外部 LLM API
- **渐进式加载优先**：路径 B 下，严禁一次性将全部段落组加载到上下文
- **保持幂等**：覆盖写入天然幂等，重复执行产生一致结果
- **覆盖输出**：AI 重写内容直接覆盖 extract 输出的原文，最终文件为单一文档（元数据 + `## 原文` + 重写稿），不保留原始转录文本。`## 原文` 二级标题统筹所有 `###` 三级子标题内容
- **中文输出**：所有重写内容和报告使用中文
- **信息完整性**：重写后的内容必须保留原文 ≥ 95% 的核心信息点
