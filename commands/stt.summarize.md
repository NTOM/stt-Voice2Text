## User Input

```text
$ARGUMENTS
```

你 **必须** 在执行前考虑用户输入（如果非空）。

## Operating Constraints

- **文件写入范围**：仅允许修改用户指定的 `.md` 文件（追加/替换第三章）
- **幂等性保证**：重复执行同一输入必须产生一致结果，不会产生重复内容
- **渐进式加载**：内容 > 6000 tokens 时自动切换分段分析策略
- **Token 预算**：单次分析窗口不超过 6000 tokens
- **错误即停**：任何步骤失败必须立即报告并停止，不得静默跳过

## Outline

### 1. 前置验证

1. 检查 `{{md-file-path}}` 是否为非空字符串
   - 为空 → 提示用户提供 MD 文件路径，**ABORT**
2. 检查文件是否存在
   - 不存在 → 提示用户检查路径，**ABORT**
3. 检查文件扩展名是否为 `.md`
   - 非 `.md` → 提示文件格式不正确，**ABORT**
4. 读取文件内容，验证是否包含 `## 第一章：原文` 标题
   - 不包含 → 提示文件格式异常，建议先运行 `/stt.json2md`，**ABORT**
5. 验证是否包含 `## 第二章：AI 重写` 标题
   - 不包含 → 提示缺少第二章，建议先运行 `/stt.json2md`，**ABORT**
6. 验证正文内容非空（第一章和第二章下有实际文本）
   - 正文为空 → 提示内容为空，**ABORT**

记录文件路径为 `TARGET_MD`。

### 2. 渐进式加载与 Token 估算

1. 提取第一章和第二章的正文内容（作为分析素材）
2. 估算 Token 数量：
   - 中文字符数 × 1.5（每字符 ≈ 1.5 tokens）
   - 英文单词数 × 1.0（每单词 ≈ 1 token）
   - **总 Token 数** = 中文 tokens + 英文 tokens
3. 记录：估算 token 数、总字符数

**阈值判定**：6000 tokens

| 条件 | 路径 | 加载策略 |
|------|------|----------|
| ≤ 6000 tokens | 路径 A（全量分析） | 一次性加载全文 |
| > 6000 tokens | 路径 B（分段分析） | 按段落组渐进加载 |

### 3. 幂等性检查

1. 搜索 `TARGET_MD` 中是否已存在 `## 第三章：内容分析`
2. **如果存在**：删除从该标题到文件末尾的所有内容（幂等替换）
3. **如果不存在**：保留原有内容，准备追加

### 4. 生成第三章（内容分析）

#### 路径 A：全量分析（≤ 6000 tokens）

对全文执行一次性分析，使用以下 prompt：

> **分析 Prompt：**
>
> 你是一位专业的内容分析师。请对以下演讲稿/文章进行深度分析，生成一份结构化的内容分析报告。
>
> ## 必选输出（必须包含）：
> 1. **主题**：用一句话概括文章的核心主题
> 2. **核心观点**：提取文章中的主要观点/论点，以编号列表形式呈现
>
> ## 可选输出（根据内容相关性选择，至少选择 1 项）：
> - **论据与案例**：当文章包含具体案例、数据或论证时
> - **关键数据**：当文章引用了重要的数字、统计或指标时
> - **争议与反思**：当文章包含辩证讨论或不同观点时
> - **行动建议**：当文章提供了可执行的建议或方法论时
> - **关键引用**：当文章中有值得摘录的精彩语句时
> - **概念解释**：当文章引入了专业术语或新概念时
> - **时间线/流程**：当文章描述了时序事件或步骤流程时
>
> ## 要求：
> - 结构要根据内容动态调整，不要生搬硬套
> - 分析要精简而清晰，类似论文摘要的风格
> - 使用中文输出
> - 每个章节标题使用 `### ` 级别的 Markdown 标题
>
> ---
>
> 以下是需要分析的文章内容：
>
> {第一章 + 第二章正文内容}

#### 路径 B：分段分析（> 6000 tokens）

**渐进式加载策略**：每次仅加载一个段落组到上下文，分析完成后释放，再加载下一组。

1. **信息性提示**（非阻塞，自动继续）：
   ```
   ℹ️ 内容较长（约 N 字 / ~T tokens），将分 M 段进行渐进式分析
   ```

2. **按语义段落分组切分**：
   - 将段落按顺序分组，每组 ≤ 4000 字符
   - **绝不在段落中间切分** — 始终在段落边界处切分
   - 记录总段数 M

3. **逐段渐进分析**（每次仅加载当前段）：

   > 这是第 X/M 段内容，请分析以下内容。使用与完整分析相同的结构（主题、核心观点、以及相关的可选章节）。
   >
   > {当前段落组内容}

4. **合并分析结果**：
   - **主题**：综合所有段的主题，提炼为一句话
   - **核心观点**：合并去重，按重要性排序
   - **可选章节**：合并同类内容，去除重复
   - 确保最终输出是一份完整的分析报告，而非拼接片段

### 5. 写入第三章并验证

1. 在 `TARGET_MD` 文件末尾追加第三章：

   ```markdown

   ---

   ## 第三章：内容分析

   {Step 4 生成的分析内容}
   ```

2. 保存文件

### 6. 完成报告

**输出验证**（自动执行，不阻塞）：

- [ ] `TARGET_MD` 文件存在且非空
- [ ] 包含 `## 第一章：原文` 标题
- [ ] 包含 `## 第二章：AI 重写` 标题
- [ ] 包含 `## 第三章：内容分析` 标题
- [ ] 第三章包含「主题」和「核心观点」必选章节
- [ ] 无重复的第三章内容

**完成报告**：

```
✅ 内容分析完成

| 指标 | 值 |
|------|-----|
| 分析文件 | {TARGET_MD 路径} |
| 分析内容字符数 | {N} 字 |
| 估算 Token 数 | {T} tokens |
| 分析路径 | 路径 A（全量） / 路径 B（分 M 段） |
| 可选章节数 | {K} 个 |

📄 文件现包含三章完整内容：
- 第一章：原文（json2md.py 脚本生成）
- 第二章：AI 重写（AI 生成的优化演讲稿）
- 第三章：内容分析（AI 生成的结构化分析报告）
```

**错误处理**：
- 验证项任一失败 → 输出失败项 + 建议修复操作，标记为 ⚠️ 部分完成

## Behavior Rules

- **绝不跳过错误**：任何步骤失败必须立即停止并报告，不得静默继续
- **渐进式加载优先**：路径 B 下，严禁一次性将全部段落组加载到上下文
- **保持幂等**：重复执行不会产生重复的第三章内容
- **保留原始格式**：不修改第一章和第二章的任何内容，仅追加/替换第三章
- **中文输出**：所有分析内容和报告使用中文
- **必选章节不可省略**：主题和核心观点必须出现在输出中
