# Data Model: JSON Voice Transcription to Markdown Article & Analysis

**Feature**: 002-json-voice-to-md
**Date**: 2026-03-31
**Updated**: 2026-03-31 (post-clarification)

## Entities

### TranscriptionSegment (Input)

A single unit of transcribed speech from the stt JSON export.

| Field | Type | Description | Validation |
|---|---|---|---|
| `line` | int | Sequential segment number (1-based) | Must be > 0 |
| `start_time` | string | Segment start in SRT format `HH:MM:SS,mmm` | Must match pattern `\d{2}:\d{2}:\d{2},\d{3}` |
| `end_time` | string | Segment end in SRT format `HH:MM:SS,mmm` | Must match pattern `\d{2}:\d{2}:\d{2},\d{3}` |
| `text` | string | Transcribed text content | Must be non-empty after strip |

### TranscriptionExport (Input)

The complete JSON file exported by stt.

| Field | Type | Description |
|---|---|---|
| (root) | array | Ordered list of `TranscriptionSegment` objects |

**Validation Rules**:
- Root must be a JSON array
- Array must contain at least 1 element
- Each element must have all 4 required fields
- Segments should be ordered by `line` number

### Paragraph (Intermediate)

A merged group of consecutive segments forming a natural text block.

| Field | Type | Description |
|---|---|---|
| `text` | string | Merged text content from multiple segments |
| `start_time` | string | Start time of the first segment in the group |
| `end_time` | string | End time of the last segment in the group |
| `segment_count` | int | Number of original segments merged into this paragraph |

**Merging Rules**:
- Merge when time gap between consecutive segments < 2 seconds
- Break on sentence-ending punctuation: 。！？.!?
- Max paragraph length: ~500 characters (safety cap)

### MarkdownArticleDocument (Output)

The generated two-chapter Markdown file.

| Section | Content | Required | Generated By |
|---|---|---|---|
| Title (H1) | Audio filename + "演讲稿与内容分析" | Yes | Python script |
| Metadata block | Source file, segment count, paragraph count, duration | Yes | Python script |
| Separator (`---`) | Visual break | Yes | Python script |
| **Chapter 1 (H2)**: 原文 | Merged paragraphs as flowing article text | Yes | Python script (rule-based) |
| Separator (`---`) | Visual break between chapters | Yes | CodeBuddy command |
| **Chapter 2 (H2)**: 内容分析 | AI-generated analysis | Yes* | CodeBuddy command (AI) |

*Chapter 2 is only generated when using the CodeBuddy command. Standalone CLI produces Chapter 1 only.

### Chapter2Analysis (Output — AI Generated)

The content analysis structure for Chapter 2.

| Section | Level | Required | Description |
|---|---|---|---|
| 主题 | H3 | **Yes** | One-sentence topic summary |
| 核心观点 | H3 | **Yes** | Numbered list of key arguments/viewpoints |
| *Dynamic sections* | H3 | At least 1 | Selected from optional pool based on content |

**Optional Section Pool**:
| Section Name | When to Include |
|---|---|
| 论据与案例 | Content contains specific examples or case studies |
| 关键数据 | Content references important numbers or statistics |
| 争议与反思 | Content includes dialectical discussion or opposing views |
| 行动建议 | Content provides actionable advice or methodology |
| 关键引用 | Content has notable quotes worth excerpting |
| 概念解释 | Content introduces technical terms or new concepts |
| 时间线/流程 | Content describes sequential events or step-by-step processes |

### ContentChunk (Intermediate — for long content)

A group of paragraphs that fits within the AI context window.

| Field | Type | Description |
|---|---|---|
| `paragraphs` | list[Paragraph] | Consecutive paragraphs in this chunk |
| `chunk_index` | int | Position in the sequence (1-based) |
| `total_chunks` | int | Total number of chunks |
| `estimated_tokens` | int | Approximate token count |

### ConversionOptions

Runtime configuration for the conversion process.

| Option | Type | Default | Description |
|---|---|---|---|
| `input_path` | string | (required) | Path to JSON file or directory |
| `output_dir` | string | same as input | Output directory for Markdown files |
| `overwrite` | bool | false | Overwrite existing output files |

## Relationships

```text
TranscriptionExport (1) ──contains──▶ (N) TranscriptionSegment
TranscriptionSegment (N) ──merges into──▶ (1) Paragraph
Paragraph (N) ──composes──▶ (1) Chapter 1 (Article)
Paragraph (N) ──groups into──▶ (1) ContentChunk (if long)
ContentChunk (N) ──analyzed by AI──▶ (1) Chapter2Analysis
Chapter 1 + Chapter2Analysis ──composes──▶ (1) MarkdownArticleDocument
ConversionOptions (1) ──configures──▶ (1) Conversion Process
```

## State Transitions

This feature has no persistent stateful entities. The conversion pipeline is:

```text
JSON File → Parse → [TranscriptionSegment] → Merge → [Paragraph] → Format → Chapter 1
                                                          ↓
                                                   Length Check
                                                    ↓         ↓
                                              (short)      (long)
                                                ↓            ↓
                                          Analyze All   Split → Analyze Each → Merge
                                                ↓                              ↓
                                            Chapter 2  ←──────────────────────┘
                                                ↓
                                     MarkdownArticleDocument
```
